## M1: 슬랙 명령어 기반 Linear 관리
> **예상 기간**: 2주  
> **의존성**: M0 완료  
> **난이도**: 중

### 배경
데일리 인터뷰 외에도, PM이 슬랙에서 자연어로 Linear를 직접 관리할 수 있어야 한다.
현재 PRD에는 "수시 이슈 생성"만 있고, 조회/수정/삭제가 빠져 있다.

### 기능
| 명령 (자연어) | 동작 |
|--------------|------|
| "내 이슈 보여줘" | 본인 할당 이슈 목록 DM 응답 |
| "이번 스프린트 남은 거" | 현재 사이클 미완료 이슈 |
| "PRODUCT-42 완료 처리해줘" | 이슈 상태 → Done |
| "PRODUCT-42 우선순위 긴급으로" | 우선순위 변경 |
| "이영희한테 PROJECT-128 할당해줘" | 담당자 변경 |
| "이번 주 프로덕트팀 현황" | 팀별 이슈 통계 요약 |

### 기술 포인트
- AI가 자연어를 파싱해서 의도(intent) + 파라미터 추출
- Linear SDK로 실행 후 결과를 슬랙 DM으로 응답
- 위험한 동작(삭제, 대량 변경)은 확인 버튼 표시

### 산출물
- `src/lib/ai/intent-parser.ts` — 자연어 → 명령 구조화
- `src/lib/slack/commands.ts` — 명령 실행 + 응답 포맷팅
- `prompts/command-parser.md` — 명령 파싱 프롬프트

## M2: Google Sheets → Linear 마이그레이션 도구
> **예상 기간**: 2주  
> **의존성**: M0 완료 (Linear 연동 모듈)  
> **난이도**: 중

### 배경
현재 SSOT는 Google Sheets. Linear로 전환하려면 기존 데이터를 이전해야 한다.
한 번에 전환하면 팀 저항이 크므로 점진적 이전 지원.

### 기능
1. **CSV 임포트**: Sheets에서 CSV 다운로드 → Linear 이슈 벌크 생성
2. **컬럼 매핑 UI**: CSV 컬럼 → Linear 필드 매핑 (제목, 담당자, 상태, 라벨 등)
3. **드라이런**: 실제 생성 전 미리보기
4. **매핑 프리셋 저장**: 같은 형태의 Sheets를 반복 임포트할 때 재사용

### 기술 포인트
- CSV 파싱 + 컬럼 자동 감지 (AI 기반)
- 담당자 이름 → Linear 유저 매칭 (fuzzy match)
- 상태값 변환 ("진행중" → "In Progress")

### 산출물
- `src/lib/migration/csv-parser.ts`
- `src/lib/migration/column-mapper.ts`
- `src/lib/migration/bulk-import.ts`
- CLI 또는 간단한 웹 UI

## M3: 주간 보고서 자동 생성
> **예상 기간**: 1.5주  
> **의존성**: M0 완료  
> **난이도**: 하

### 배경
PM 리더는 매주 팀 현황을 보고해야 한다. Linear 데이터 + 데일리 요약을 조합하면 자동 생성 가능.

### 기능
1. **주간 요약 자동 생성**: 금요일 오후 → 한 주간 완료/진행/지연 이슈 집계
2. **팀원별 기여도**: 완료 이슈 수, 새로 생성된 이슈 수
3. **다음 주 예고**: 마감 임박 이슈, 블로커 목록
4. **슬랙 + Linear 동시 배포**: 슬랙 채널에 포스팅 + Linear Project Update로 등록

### 포맷 (자동 생성)
```markdown
## 주간 보고 (2026-04-07 ~ 2026-04-11)

### 완료된 작업
- [PRODUCT-42] 온보딩 플로우 스펙 정리 (김한솔)
- [PROJECT-128] Q2 대시보드 디자인 (이영희)

### 진행 중
- [PRODUCT-45] 결제 페이지 개선 — 60% (박철수)

### 블로커
- [PROJECT-130] 백엔드 API 스펙 대기 중 (담당: 백엔드팀)

### 다음 주 주요 일정
- 4/14 (월) PRODUCT-45 마감
- 4/16 (수) Q2 기획 리뷰 미팅
```

### 산출물
- `src/lib/reports/weekly.ts`
- `templates/공통/주간보고.md` (이미 PRD에서 생성)
- Cron: 매주 금요일 17:00 KST

## M4: 슬랙 이모지 → 이슈 생성
> **예상 기간**: 1주  
> **의존성**: M0 완료  
> **난이도**: 하

### 배경
사용자가 예전 회사에서 경험한 기능. 슬랙 메시지에 특정 이모지를 달면 자동으로 Linear 이슈가 생성되는 방식.

### 기능
1. 슬랙 메시지에 📋 (또는 지정 이모지) reaction 추가 → 이슈 생성
2. AI가 메시지 내용 분석 → 제목/본문/라벨 자동 채움
3. 원본 메시지에 스레드로 "이슈 생성됨: PROJECT-xxx" 링크 응답
4. 이모지 종류별 다른 동작 가능 (📋=이슈, 🐛=버그, ⚡=긴급)

### 기술 포인트
- Slack `reaction_added` 이벤트 구독
- 원본 메시지 내용 가져와서 AI 분석
- 이미 이슈가 생성된 메시지인지 중복 체크

### 산출물
- `src/lib/slack/reactions.ts`
- Event 핸들러에 reaction 라우팅 추가

## M5: Linear Webhook → 슬랙 알림
> **예상 기간**: 1주  
> **의존성**: M0 완료  
> **난이도**: 하

### 배경
Linear에서 변경이 일어났을 때 슬랙으로 알림. 현재는 Linear → Slack 방향이 없다.

### 기능
1. **담당자 변경**: "PRODUCT-42가 당신에게 할당되었어요" DM
2. **코멘트 알림**: 내 이슈에 코멘트 달리면 DM
3. **상태 변경**: 팀 채널에 "PRODUCT-42: In Progress → Done" 알림
4. **마감 임박**: D-1, D-day 자동 리마인더
5. **커스텀 필터**: 팀원별로 어떤 알림을 받을지 설정

### 기술 포인트
- Linear Webhook 수신 엔드포인트 (`/api/linear/webhook`)
- Webhook payload → 알림 메시지 변환
- 유저 매핑: Linear User ID ↔ Slack User ID

### 산출물
- `src/app/api/linear/webhook/route.ts`
- `src/lib/linear/notifications.ts`
- `src/lib/config/notification-preferences.ts`

## M6: 대화형 스프린트 플래닝
> **예상 기간**: 2주  
> **의존성**: M1 (명령어 기반 관리)  
> **난이도**: 상

### 배경
격주 스프린트 플래닝을 슬랙에서 AI 퍼실리테이터와 함께 진행.

### 기능
1. 플래닝 시작 명령 → AI가 백로그에서 후보 이슈 추천
2. 팀원별 캐파 확인 ("이번 스프린트 몇 포인트 가능?")
3. 이슈 할당 제안 → 팀원 승인/거절
4. 확정된 이슈를 현재 사이클에 자동 등록
5. 플래닝 결과 요약 채널에 포스팅

### 기술 포인트
- Linear Cycle API 활용
- 이슈 포인트/복잡도 기반 추천
- 멀티 유저 대화 (슬랙 채널 스레드에서 진행)

### 산출물
- `src/lib/planning/facilitator.ts`
- `src/lib/planning/capacity.ts`
- `src/lib/planning/assignment.ts`
- `prompts/sprint-planning.md`

## M7: 회의록 자동 이슈 변환
> **예상 기간**: 1.5주  
> **의존성**: M0 완료  
> **난이도**: 중

### 배경
PM 미팅 후 회의록을 슬랙에 공유하면, AI가 액션 아이템을 추출해서 Linear 이슈로 만드는 기능.

### 기능
1. 슬랙 채널에 회의록 텍스트 붙여넣기 + 봇 멘션
2. AI가 액션 아이템 추출 (담당자, 마감일, 내용)
3. 이슈 목록 프리뷰 → 확인 후 벌크 생성
4. 회의록 원문은 Linear 이슈에 링크로 첨부

### 기술 포인트
- 긴 텍스트 처리 (회의록은 길 수 있음 → Sonnet 사용)
- 담당자 이름 → Slack/Linear 유저 매칭
- 회의록 형태 다양성 대응 (자유 형식, 불릿 포인트, 마크다운 등)

### 산출물
- `src/lib/ai/meeting-notes.ts`
- `templates/공통/회의록.md`

## M8: 주간 만족도 인터뷰
> **예상 기간**: 1주  
> **의존성**: M0 완료 (DM 인터뷰 인프라 재사용)  
> **난이도**: 하

### 배경
봇 도입 후 팀원들의 실제 만족도와 개선 요청을 정기적으로 수집해야 한다.
설문 링크 방식은 응답률이 낮으므로, 데일리 인터뷰와 같은 DM 대화 방식으로 진행.

### 기능
1. **매주 금요일 오후 3시**(15:00 KST) 각 팀원에게 DM 자동 발송
2. AI가 1:1 인터뷰 진행 (버튼 평점 + 자연어 피드백)
3. 피드백 수집 → 구조화 저장
4. PM 리더에게 주간 만족도 리포트 자동 DM

### 인터뷰 플로우
```
봇: "이번 주 수고했어요! 이번 주 봇 사용 경험은 어땠어요?
     점수로 먼저 골라주세요."

     [😡 1] [😕 2] [😐 3] [😊 4] [🤩 5]

유저: [😊 4] 클릭

봇: "좋아요! 혹시 이번 주에 불편했거나 개선됐으면 하는 부분이 있었어요?"

유저: "이슈 자동 생성은 좋은데, 가끔 템플릿이 잘못 매칭돼요"

봇: "어떤 상황에서 잘못 매칭됐는지 예시를 알려줄 수 있어요?"

유저: "버그 리포트인데 기획서 템플릿이 적용된 적 있어요"

봇: "피드백 감사합니다! 좋은 주말 보내세요 👋"
```

### 리포트 (PM 리더에게 DM)
```
📊 주간 만족도 리포트 (4/7 ~ 4/11)
─────────────────────────
평균 점수: 4.2 / 5.0 (지난주 3.8 대비 ↑)
응답률: 9/10명

긍정 키워드: 이슈 자동 생성, 시간 절약
개선 요청:
 • 템플릿 매칭 오류 — 버그↔기획서 혼동 (2명)
 • follow-up 질문 줄여달라 (1명)

추세: 3주 연속 상승 (3.5 → 3.8 → 4.2)
```

### 기술 포인트
- M0의 DM 인터뷰 인프라 그대로 재사용 (Cron + DM + AI + KV)
- Slack Interactive 버튼으로 평점 수집
- 3점 이하 → AI follow-up 질문 (상세 피드백)
- 4점 이상 → 간단히 "개선점 있어요?" 1회만 질문
- 주차별 점수 추적 → 추세 분석

### Cron 설정
```json
{
  "path": "/api/cron/satisfaction",
  "schedule": "0 6 * * 5"
}
```
> UTC 6:00 금요일 = KST 15:00 금요일

### 산출물
- `src/lib/ai/satisfaction-interview.ts`
- `src/lib/reports/satisfaction.ts`
- `prompts/satisfaction-interview.md`
- 만족도 데이터 저장 스키마 (KV 또는 DB)

## 마일스톤 타임라인 개요

```
       Week 1-3        Week 4-5       Week 6-7       Week 8+
      ┌─────────┐    ┌──────────┐   ┌──────────┐   ┌──────────┐
      │ M0      │    │ M1       │   │ M3       │   │ M6       │
      │ 데일리  │───▶│ 슬랙     │   │ 주간보고 │   │ 스프린트 │
      │ 스크럼봇│    │ 명령어   │   │ 자동생성 │   │ 플래닝   │
      └─────────┘    └──────────┘   └──────────┘   └──────────┘
           │          ┌──────────┐   ┌──────────┐   ┌──────────┐
           ├────────▶│ M2       │   │ M4       │   │ M7       │
           │         │ Sheets   │   │ 이모지   │   │ 회의록   │
           │         │ 마이그레 │   │ →이슈    │   │ →이슈    │
           │         └──────────┘   └──────────┘   └──────────┘
           │                        ┌──────────┐
           └───────────────────────▶│ M5       │
                                    │ Webhook  │
                                    │ →알림    │
                                    └──────────┘

  ■ 필수 순서: M0 → M1 → M6
  ■ M0 이후 병렬 가능: M2, M3, M4, M5, M7, M8
  ■ 우선순위 추천: M0 → M4 → M8 → M1 → M5 → M3 → M7 → M2 → M6
```

### 우선순위 추천 근거

| 순위 | 마일스톤 | 이유 |
|------|---------|------|
| 1 | **M0** 데일리 스크럼 봇 | 핵심 MVP, 팀 도입의 시작점 |
| 2 | **M4** 이모지 → 이슈 | 구현 간단(1주), 팀 체감 효과 큼, 사용자가 이전에 경험한 기능 |
| 3 | **M8** 주간 만족도 인터뷰 | M0 인프라 재사용, 도입 초기 피드백 수집이 중요 |
| 4 | **M1** 슬랙 명령어 | DM으로 조회/수정까지 가능해야 "Linear 안 열어도 됨" 완성 |
| 5 | **M5** Webhook 알림 | 양방향 연동 완성 (슬랙→Linear + Linear→슬랙) |
| 6 | **M3** 주간 보고서 | PM 리더 업무 자동화, 보고 부담 경감 |
| 7 | **M7** 회의록 → 이슈 | PM 미팅 후 반복 작업 제거 |
| 8 | **M2** Sheets 마이그레이션 | 전환 완료 시점에 1회성, 급하지 않음 |
| 9 | **M6** 스프린트 플래닝 | 복잡도 높음, 팀이 Linear에 익숙해진 후 도입 |
