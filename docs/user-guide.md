# Linear 슬랙봇 사용 가이드

슬랙에서 대화하거나 이모지를 다는 것만으로 Linear에 태스크가 자동으로 만들어지는 봇입니다.
Linear를 직접 열지 않아도 됩니다.

---

## 시작하기

### 봇 찾기

1. 슬랙 좌측 사이드바에서 **앱** 섹션을 찾습니다
2. **task-tracking-bot** 이 보이면 바로 사용 가능합니다
3. 안 보이면 **앱 추가** 를 클릭하고 `task-tracking-bot` 을 검색해서 추가합니다

<!-- 스크린샷: 슬랙 앱 섹션에서 task-tracking-bot 찾는 모습 -->

### 봇에게 DM 보내기

1. 앱 목록에서 **task-tracking-bot** 을 클릭합니다
2. **메시지** 탭을 선택합니다
3. 아무 메시지나 보내보세요. 봇이 응답하면 준비 완료입니다

### 채널에 봇 추가하기

이모지로 태스크를 만들려면 봇이 해당 채널에 있어야 합니다.

1. 봇을 추가하고 싶은 채널에 들어갑니다
2. 메시지 입력창에 `/invite @task-tracking-bot` 을 입력하고 전송합니다
3. "task-tracking-bot이 채널에 참여했습니다" 메시지가 뜨면 완료입니다

## 1. 아침 데일리 스크럼

매일 아침 7시에 봇이 DM으로 찾아옵니다.

> 봇: 좋은 아침이에요! 오늘 할 일에 대해 간단히 알려주세요.

그냥 평소처럼 말하면 됩니다:

> 나: 오늘 Adobe 배너 작업 마무리하고, 오후에 Hecto 미팅 있어요. 배너는 오늘까지 납품이에요.

봇이 한두 번 더 물어볼 수 있어요 (마감일, 블로커 등). 대화가 끝나면:

- **Linear에 태스크 자동 생성** (제목, 설명, 라벨, 마감일, 담당자 전부 자동)
- **데일리 스크럼 채널에 요약 포스팅** (스레드에 실시간으로 올라감)

> 봇: 감사합니다! 좋은 하루 보내세요.
> Linear에 등록된 이슈: PROJ-48, PROJ-49

## 2. 이모지로 태스크 만들기

슬랙에서 메시지에 이모지를 달면 자동으로 태스크가 만들어집니다.

### 사용 방법

1. 태스크로 만들고 싶은 메시지를 찾습니다
2. 해당 메시지에 이모지 리액션을 답니다:

| 이모지 | 검색명 | 용도 |
|--------|--------|------|
| ![emoji-task](../public/emoji-task.png) | `emoji-task` | 일반 태스크 |
| 🐛 | `bug` | 버그 리포트 (높은 우선순위) |
| ⚡ | `zap` | 긴급 태스크 (최고 우선순위) |

3. 봇이 스레드에 자동으로 답변합니다:

> 봇: Linear에 등록했어요: **PROJ-50** — Adobe 배너 디자인 수정 요청

<!-- 스크린샷: 메시지에 이모지 달고 봇이 스레드에 응답한 모습 -->

### 어떤 채널에서든 가능합니다

- `project_` 로 시작하는 채널 → 프로젝트팀에 태스크 생성
- `product_` 로 시작하는 채널 → 프로덕트팀에 태스크 생성

### 봇이 채널에 없다면?

채널에서 `/invite @task-tracking-bot` 을 입력해서 봇을 추가하세요.

## 4. 중복 태스크 방지

비슷한 태스크가 이미 있으면 봇이 대화로 먼저 확인합니다.

> 봇: 비슷한 이슈가 이미 있어요:
> **PROJ-42**: Adobe 배너 디자인 작업
>
> 새로 만들려는 이슈: Adobe 배너 작업 마무리
>
> 기존 이슈를 업데이트할까요, 새로 만들까요?

> 나: 기존 거 업데이트해줘

> 봇: **PROJ-42** 에 업데이트했어요.

대답은 자유롭게 하면 됩니다:
- "업데이트해줘" / "기존 거" → 기존 이슈에 코멘트 추가
- "새로 만들어줘" → 새 태스크 생성
- "건너뛰기" / "됐어" → 아무것도 안 함

## 5. 자동으로 채워지는 것들

봇이 태스크를 만들 때 자동으로 설정하는 항목:

| 항목 | 설명 |
|------|------|
| **제목** | 대화 내용에서 핵심을 추출 |
| **설명** | 배경, 할 일, 완료 조건이 포함된 상세 설명 |
| **팀** | 채널명 또는 대화 내용으로 자동 판단 (프로젝트팀/프로덕트팀) |
| **프로젝트** | 클라이언트명을 감지하여 자동 연결 (Adobe KR, Hecto 등) |
| **담당자** | 메시지를 보낸 사람으로 자동 지정 |
| **상태** | Todo로 설정 |
| **우선순위** | 대화 맥락에서 판단 (긴급/높음/보통/낮음) |
| **라벨** | 내용에 따라 자동 분류 (클라이언트요청, 미팅, 버그 등) |
| **마감일** | "오늘까지", "이번 주" 같은 표현을 날짜로 변환 |
| **Estimate** | 작업 크기 자동 판단 (1/2/3/5 포인트) |

## 자주 묻는 질문

**Q: 봇이 아침에 DM을 안 보내요**
A: 봇은 등록된 팀원에게만 DM을 보냅니다. 관리자에게 본인 추가를 요청하세요.

**Q: 이모지를 달았는데 반응이 없어요**
A: 봇이 해당 채널에 멤버로 추가되어 있어야 합니다. 채널에서 `/invite @task-tracking-bot` 을 입력하세요.

**Q: 태스크가 잘못 만들어졌어요**
A: Linear에서 직접 수정하거나, 봇에게 DM으로 상태/우선순위/마감일을 변경할 수 있습니다.

**Q: DM으로 말했는데 인터뷰처럼 질문을 해요**
A: "PROJ-42 완료해줘" 같은 명령은 바로 실행됩니다. "오늘 할 일"처럼 업무 내용을 말하면 인터뷰 모드로 진입합니다.

**Q: 이미 있는 태스크를 또 만들려고 해요**
A: 비슷한 태스크가 있으면 대화로 먼저 확인합니다. "업데이트해줘"라고 답하면 중복 없이 기존 이슈에 코멘트만 추가됩니다.
